# Git
1. Названия файлов в решении и проектах лучше давать на английском языке. На сервере может не оказаться русской локализации.
2. Используйте конвенцию (правила) по наименованию, проектов, решений. Более подробно можно прочитать тут:
https://www.calabonga.net/blog/post/naming-for-projects-in-solution


## Для того, чтобы ваша работа была ещё лучше, я рекомендую:
1.Разделите репозитарий по задачам т.е одна задача == один репозитарий. 
Если в одном модуле несколько задач нужно выделить каждую задачу в отдельный проект.
2.Это часть нашей работы разбирать (делать код ревью) чужой код, возможно, что то для себя новое подчеркнуть.
